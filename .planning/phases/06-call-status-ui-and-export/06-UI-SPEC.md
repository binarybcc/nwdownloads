# Phase 6 — UI Design Contract

> Visual and interaction contract for the Call Status UI and Export phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property          | Value                                                  |
| ----------------- | ------------------------------------------------------ |
| Tool              | none                                                   |
| Preset            | not applicable                                         |
| Component library | none (vanilla JS with inline styles)                   |
| Icon library      | Unicode emoji (existing pattern)                       |
| Font              | System default (existing), monospace for IDs and dates |

**Note:** This project uses Tailwind CSS for layout (cards, grids) but the `SubscriberTablePanel` renders entirely with inline styles. Phase 6 follows the inline style pattern for all table modifications, consistent with the existing codebase.

---

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage                                                        |
| ----- | ----- | ------------------------------------------------------------ |
| xs    | 4px   | Left border width on table rows                              |
| sm    | 8px   | Cell padding vertical (0.35rem ~ 5.6px, existing), icon gaps |
| md    | 16px  | Section padding, panel content margins                       |
| lg    | 24px  | Panel header padding (p-6 existing)                          |
| xl    | 32px  | Panel slide-out gap from edge                                |
| 2xl   | 48px  | Not used in this phase                                       |
| 3xl   | 64px  | Not used in this phase                                       |

Exceptions:

- Status icon container: 28px width/height (not a multiple of 4, but matches existing badge sizing pattern in the table)
- Cell padding: 0.35rem vertical / 0.5rem horizontal (existing pattern, preserved for consistency)

## Color

### Surface Colors (unchanged from existing)

| Role            | Value                | Usage                                                           |
| --------------- | -------------------- | --------------------------------------------------------------- |
| Dominant (60%)  | `#FFFFFF`            | Table row backgrounds (odd rows)                                |
| Secondary (30%) | `#F0FDFA` (teal-50)  | Table row backgrounds (even rows, existing alternating pattern) |
| Accent (10%)    | `#14B8A6` (teal-500) | Table header background, panel header gradient                  |

### Call Status Colors (new for this phase)

These colors are used in two contexts: browser table (left border + icon background) and XLSX export (row fill).

#### Browser Table

| Status                      | Left Border         | Icon Background                        | Hex       |
| --------------------------- | ------------------- | -------------------------------------- | --------- |
| Contacted (placed outbound) | 4px solid `#22C55E` | `#22C55E` at 12% opacity (`#22C55E20`) | green-500 |
| Inbound (received/missed)   | 4px solid `#F59E0B` | `#F59E0B` at 12% opacity (`#F59E0B20`) | amber-500 |
| No contact                  | 4px solid `#EF4444` | `#EF4444` at 12% opacity (`#EF444420`) | red-500   |

**Left border:** Applied via inline `border-left: 4px solid {color}` on each `<tr>`.
**Icon background:** Circular span (28px, border-radius 50%) behind the phone emoji.
**Row background:** Unchanged. White/alternating teal-50 pattern preserved. No full row tint in browser.

#### XLSX Export Row Fills

| Status                    | Fill Hex (no #) | CSS Hex   | Print Contrast                     |
| ------------------------- | --------------- | --------- | ---------------------------------- |
| Contacted (placed)        | `DCFCE7`        | `#DCFCE7` | Light green, distinct gray in B&W  |
| Inbound (received/missed) | `FEF9C3`        | `#FEF9C3` | Light yellow, distinct gray in B&W |
| No contact                | `FEE2E2`        | `#FEE2E2` | Light red, distinct gray in B&W    |

**XLSX fills replace the existing alternating teal pattern.** Status color takes priority over alternating rows in the export.

#### XLSX Header Styling

| Element                | Style                                                                 |
| ---------------------- | --------------------------------------------------------------------- |
| Timestamp row (Row 1)  | Italic, 10pt, `#666666` text, left-aligned, merged across all columns |
| Column headers (Row 2) | Bold, 12pt, white text on `#0891B2` background (existing pattern)     |
| Data rows (Row 3+)     | Status-based fill per table above                                     |

Accent reserved for: table header row background, panel header gradient, account ID link color, email link color (all existing uses preserved).

## Copywriting Contract

| Element                      | Copy                                                                                                             |
| ---------------------------- | ---------------------------------------------------------------------------------------------------------------- |
| Column header                | "Status"                                                                                                         |
| Tooltip: contacted           | "{Direction} \u2022 {Mon DD}, {H:MM AM/PM} \u2022 {Agent}" -- Example: "Placed \u2022 Mar 18, 2:15 PM \u2022 BC" |
| Tooltip: no contact          | "No contact recorded"                                                                                            |
| Sort arrow (default)         | Down arrow indicator in Status header (not-contacted first)                                                      |
| Sort arrow (toggled)         | Up arrow indicator in Status header (contacted first)                                                            |
| Sync timestamp (browser)     | "Call data as of {Mon DD, YYYY H:MM AM/PM}" -- Example: "Call data as of Mar 20, 2026 2:02 PM"                   |
| Sync timestamp (XLSX Row 1)  | "Call data as of {YYYY-MM-DD HH:MM}" -- Example: "Call data as of 2026-03-20 14:02"                              |
| XLSX column: Call Status     | Text values: "Placed", "Received", "Missed", or empty string                                                     |
| XLSX column: Last Contact    | Date/time string or empty string                                                                                 |
| XLSX column: Agent           | "BC" or "CW" or empty string                                                                                     |
| Empty state (no subscribers) | Existing empty state preserved (no change)                                                                       |
| Error state (API failure)    | Existing error handling preserved (no change)                                                                    |

**No destructive actions in this phase.** Export and sort are non-destructive operations.

### Interaction States

| Interaction                  | Behavior                                                                                                    |
| ---------------------------- | ----------------------------------------------------------------------------------------------------------- |
| Hover on phone icon          | Native `title` tooltip appears with call details                                                            |
| Hover on table row           | Background changes to `#CCFBF1` (existing behavior preserved)                                               |
| Click Status header          | Toggle sort direction (not-contacted first <-> contacted first)                                             |
| Click Status header (visual) | Arrow indicator flips direction                                                                             |
| Initial table load           | Sorted by status priority (red=0, orange=1, green=2 ascending), sub-sorted by expiration date soonest first |

## Sort Priority Mapping

| Priority Value | Status                  | Color  | Meaning                 |
| -------------- | ----------------------- | ------ | ----------------------- |
| 0 (top)        | `null` (no call_status) | Red    | No contact recorded     |
| 1              | `received` or `missed`  | Orange | Subscriber called us    |
| 2 (bottom)     | `placed`                | Green  | Staff called subscriber |

Default sort: ascending (0 on top = not-contacted first).
Toggle: descending (2 on top = contacted first).
Sub-sort within each group: `expiration_date` ascending (soonest first), always.
