Add the following purge block to web/fetch_call_logs.php AFTER the existing "=== Done." log_msg line (~line 189) and BEFORE `exit(0)` (~line 190):

```php
// Purge old call logs (90-day retention)
try {
    $purgeStmt = $pdo->exec("DELETE FROM call_logs WHERE call_timestamp < DATE_SUB(NOW(), INTERVAL 90 DAY)");
    log_msg("Purged {$purgeStmt} call_logs records older than 90 days");
} catch (\PDOException $e) {
    log_msg("WARNING: Call log purge failed: " . $e->getMessage());
}
```

Key implementation details:
- Use `$pdo->exec()` (not `$pdo->prepare()`) because there are no parameters -- `INTERVAL 90 DAY` is a SQL literal. `exec()` returns affected row count directly.
- Wrap in try/catch per Claude's discretion decision. The purge is non-critical (scraped copies of BroadWorks data), so a purge failure should log a warning but NOT prevent the script from exiting successfully.
- The `$pdo` variable is already in scope at this point in the script (used throughout the import loop above).
- Use `\PDOException` with backslash since the script is not in a namespace.
- The log_msg format matches the existing pattern: descriptive text with variable interpolation.

- Use `imported_at` column (locked decision: use `call_timestamp`)
- Add an abort threshold or row count guard (locked decision: simple DELETE, no guard)
- Move the purge before the import (locked decision: purge AFTER import completes)
